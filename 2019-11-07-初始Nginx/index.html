基础配置

Nginx指令和指令块

简单指令：由名字和参数组成，名字和参数之间用空格分隔，以分号结尾 Main 1..就是简单指令

指令块：由名称和大括号{}内的附加指令组成，不以分号结尾

在这上面的配置中http是全局参数，对整体产生影响。

server块是虚拟主机，主要对指定的主机和端口进行配置。

location块在虚拟主机下根据请求URI进行配置，URI即去掉参数的URL

有些简单指令可以存在上面三个指令块中，如果同一个简单指令同时存在的话，以最小块里面的参数为主

与客户端有关的配置主要在http块中设置。

Server块

server块是虚拟主机部分，如果请求中的host头和server_name相匹配，则将请求指向对应的server块。

server_name支持配置多个

配置规则：

精确的名字

以*开头的最长通配符名称 如 *.testnginx.com

以结尾的最长通配符名称 如estnginx.

按照文件顺序，第一个匹配到的正则表达式

如果没有匹配到server_name则会访问default_server

location块

location块是在server块中使用，当客户端请求的一个url定位不同的应用。这就需要使用请求的url匹配对应的location指令

一些特殊的指令

alias定义指定位置的替换

如果匹配到/a/test.json进入到这个location块后，会将请求变成/c/x/a/test.json

##include的使用

include用来指定主配置文件包含的其他扩展配置文件，可以用在全局参数、location块、server块中等任何一个位置。

【可以将相同的数据提取出来，从而精简配置】

常见配置注解

内置变量

$scheme 请求协议

强化基础配置

限制IP地址的访问

allow允许IP地址或IP字段访问

deny 禁止IP地址或IP字段访问

具体配置如下

禁止全部server_name访问 在http块中配置deny deny ip

禁止某个server_name访问 在server块中配置deny deny ip

禁止某个Location访问 在Location配置deny deny ip

proxy代理

proxy_pass请求代理规则

语法 proxy_pass URL;

环境 location、if in locaiton、limit_except

含义 将请求代理到后端服务器，设置后端服务的IP地址，端口号以及HTTP/HTTPS

将uri为/test的请求代理到127.0.0.1上，端口号81 使用http

代理过程中url会有以下几种变化

减少后端服务器的网络开销

有时候的请求只跟url的参数有关，并不需要读取请求头和请求体。

设置下面两个指令，将其配置成off

proxy_pass_request_body

proxy_pass_request_headers

upstream使用手册

它为反向代理提供了负载均衡及故障转移等重要功能

代理多态服务器

rewrite使用手册

完成内部重定向的功能。

内部重定向是在Nginx内部发送请求的操作，它可以将请求转发到其他location或者是对Url进行修改，而不必通过HTTP连接请求，整个过程高效。
