前言

SQL语法的学习

定义

SQL是结构化查询语句的缩写，用来访问和操作数据库系统。

hhhh我知道你们会问什么是结构化查询语句：

Structured Query Language is a standard computer language for relational database management and data manipulation.(好吧用来操作和访问关系型数据库的语句)

关系模型

some keys:

  1. 通常情况下，字段应该避免允许为NULL，可以简化查询条件，加快查询速度

三种关系 【班级表 学生表】

​ 一对多的关系：一个班级对应着多个学生

​ 多对一的关系：多个学生对应着一个班级

​ 一对一的关系：一个班主任老师对应着一个班级

主键some kyes
  1. 对于关系数据库，必须要求能够通过某个字段主键唯一区分出不同的记录。
  2. 作为主键最好是完全业务无关的字段(例如 身份证 手机号 邮箱 都不可以作为主键)，常见的可以用自增整数类型、全局唯一GUID类型
外键some keys
  1. 在students表中，通过class_id的字段，可以把数据与一张表关联起来，这种列称为外键
  2. 可以通过外键约束(无法插入class表中没有的id),不过这回影响性能，所以可以用应用逻辑来解决这个问题
  3. 多对多关系：通过一个中间表实现两个一对多的关系

更新，趁着复习间隙，入个门

查询

基本查询

该语句的作用，就是查询<表>的所有行，具体分析

  1. select 是关键字，表示要执行一个查询
  2. 表示*所有列**
  3. from表示从哪个表进行查询
条件查询

该语句的作用是进行条件查询，where后面是条件,具体分析

  1. where关键词后面带着条件例如 score>=80等等，其中score是表中的某一列
  2. 将条件表达式进阶 <条件1> and <条件2>;<条件1> or <条件2>;not <条件>
  3. 不加括号的优先级是 not and or

更高级的 like

投影查询

该语句的作用就是不是返回满足条件的全部信息，只返回一部分列，具体分析

  1. 通过 select 列1…其中注意列的名字需要与数据库同一字段，这种操作成为投影查询

进阶

该语句的作用就是，在取回的时候，显示列名是用别名，相当于在结果集中对列名进行重命名

排序

该语句的作用就是，在结果集自己定义一种排序，原本默认的排序是以Id从小到大，具体分析

  1. order by 列，是以这个列进行从小到大的排序

进阶，从高到低排序

desc的关键字表示倒序，即从大到小排序。

该语句的作用，就是先通过列1进行倒序排序，如果列1中有相同的数据，则再按列2排序

注意

如果有where子句(即条件句)，要把order by子句放到where 后面

分页查询

分页操作实际上就是从结果集截取出部分数据

分析一下

  1. limit 这里的M表示最多显示多少条数据，即一般设置为pageSize
  2. offset 这的n 表示从哪里开始，记住数据库中的索引是从0开始的，如果n值设置大于结果集的值，并不会报错，只是返回一个空集
聚合查询

即要计算表中的数据，使用聚合函数进行查询的就是聚合查询

该语句的表示查询所有列的函数，返回一个二维表，只是该二维表只有一行一列，并且列名是count(*)。

该语句就是给这个二维表的列名取了一个别名num

进阶，加上条件语句，我们可以很容易统计出数据库中的字段

分组聚合

该语句的作用，就是将结果集按照group by 列 分组，然后统计每一组中的数量

多表查询

该语句表示的是，从表1，表2中查询，结果集是这两个表数据的“乘积”，即表1与表2的每一行都两两拼在一起返回。结果集的列数是两个表的列数之和，行数是两个表的之积。这种查询又称为笛卡尔查询(是不是很熟悉)
