####配置单数据源

废话不多说，直接进入学习

因为我们做的是一个配置数据源(数据的来源),所以我们需要一个DataSource对象，如果我们要做sql操作，则需要一个JdbcTemplate用来做sql操作

现在我们来解决一些问题：

  • We’re using Spring’s @SpringBootApplication annotation on our main class to enable auto-configuration

    使用这个注解让springboot自动帮助我们配置一些东西

  • This class also implements Spring’s CommandLineRunner interface.CommandLineRunner is a simple Spring Boot interface with a run method.Spring Boot will automatically call the run method of all beans implementing this interface after the application context has been loaded

    这个接口CommandLineRunner只有一个简单的方法Run，当Spring Boot加载完应用上下文的时候会自动执行这个方法

  • JdbcTemplate这个类稍微复杂点，不过现在只要记住，要执行sql语句需要使用JdbcTemplate

  • @Autowired&@Bean

@Autowired says’please give me an instance of this class,for example,one that I created with an @Bean annotation earlier

@Bean tells Spring ‘here is an instance of this class,please keep hold of it and give it back to me when I ask’

所以要使用@Autowired前我们需要先告知Spring这个类是一个实例(@Bean)

  • endpoints这里我们得先谈到Actuator

    Actuator：Spring Boot provides actuator to monitor and manage our application.Actuator is a tool which has HTTP endpoints.when application is pushed to production,you can choose to manage and monitor your application using HTTP endpoints

    Actuator可以帮助你监控和管理SpringBoot应用，所有的这些特性都是通过HTTP endpoints来获得

    Actuator创建了所谓的endpoint来暴露HTTP或者JMX来监控和管理应用

    1. /health的endpoint提供了关于应用健康的基础信息
    2. /metrics endpoint展示了几个有用的度量信息，比如JVM内存使用情况，系统CPU使用情况、打开文件等等

    所以为了暴露HTTP我们首先得把endpoints暴露出去

  • A .properties file is a simple collection of key-value pairs that can be parsed by java.util.Properties class.

    Properties files are widely used for many purposes in all kinds of JAVA application,often to store configuration or localization data

    .properties文件通过key-value来存储输出。

配置多数据源

tips:

  1. 不同数据源的配置要分开
  2. 关注每次使用的数据源，需要自己判断使用哪个数据源，系统无法自动帮助你判断

SpringBoot两种方法

  1. 配置@Primary类型的Bean–这个将作为主要的Bean
  2. 排除SpringBoot的自动配置

这里我们先使用第二种方法

先来解决一些问题：

  • Bean:是一个Java类，有Spring容器创建并且托管的实例，Java应用是由一个个组件组成，就像是咖啡是由一个个豆子组成，所以Bean就是类的意思

  • ConfigurationProperties(“foo.datasource”)先来看这个，这个注解是为了让.properties文件里的foo.datasource对创建出来的DataSourceProperties进行赋值
